**buildPrompt helper** — constructs the body sent to `/api/generate-node`:
```typescript
function buildBody(pathId: PathId): string {
  const pathState = paths[pathId];
  const nodes = pathState?.nodes ?? [];
  return JSON.stringify({
    pathId,
    depth: nodes.length + 1,
    chart,
    previousNodes: nodes,
    griefContext: griefEntries.length > 0 ? griefEntries : undefined,
  });
}
```

**handleExpand** — called by PathColumn's onExpand prop:
```typescript
async function handleExpand(pathId: PathId) {
  const completion = completions[pathId];
  if (!completion) return;
  // `complete` from useCompletion takes a prompt string — we pass empty string
  // because all context is in the request body via the `body` option.
  // BUT useCompletion doesn't support dynamic body per-call directly.
  // SOLUTION: Pass body as the prompt encoded as JSON — route handler reads req.text() and parses.
  // Actually: use completion.complete('') with body option set at hook init time.
  // CORRECT approach per Vercel AI SDK v3: pass `body` as additional fields at call time:
  await completion.complete('', {
    body: JSON.parse(buildBody(pathId)) as Record<string, unknown>,
  });
}
```

**onFinish callback pattern** — `useCompletion` does not have a built-in `onFinish` callback in all versions. Instead, use a `useEffect` to watch `isLoading` transitioning from true to false and `completion` being non-empty, then commit the node to the store:

```typescript
// For each path, watch completion state and commit when streaming finishes
const prevLoadingRef = {
  duty: useRef(false),
  desire: useRef(false),
  transformation: useRef(false),
};

// Commit duty node
useEffect(() => {
  const c = dutyCompletion;
  if (prevLoadingRef.duty.current && !c.isLoading && c.completion) {
    const pathState = paths['duty'];
    const newNode: PathNode = {
      id: nanoid(),
      depth: (pathState?.nodes.length ?? 0) + 1,
      content: c.completion,
      isStreaming: false,
    };
    setPathNode('duty', newNode);
  }
  prevLoadingRef.duty.current = c.isLoading;
}, [dutyCompletion.isLoading, dutyCompletion.completion]);

// Repeat identically for desire and transformation paths
// (copy the useEffect block twice, substituting 'desire' and 'transformation')
```

**Three useRef calls must be at top level** — not inside objects. Rewrite as:
```typescript
const dutyPrevLoading = useRef(false);
const desirePrevLoading = useRef(false);
const transformationPrevLoading = useRef(false);
```

Then in each useEffect, reference the appropriate ref directly.
