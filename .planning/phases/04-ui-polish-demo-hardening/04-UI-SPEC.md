## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, depth dot spacing (`gap-1` = 4px or `gap-2` = 8px for dot rows — both valid) |
| sm | 8px | Compact inline padding, button vertical padding |
| md | 16px | Default element spacing, grid gaps within forms |
| lg | 24px | Column gaps in the 3-column grid (`gap-6`), section headings to content |
| xl | 32px | Layout section breaks (e.g. PathTreeView top margin `mt-8`) |
| 2xl | 48px | Major section separations (grief archive card gaps `gap-8` ≈ 32px; allow up to 48px for visual breathing room between archive entries) |
| 3xl | 64px | Page-level top/bottom padding (`py-8` = 32px on each side; 64px total vertical breathing room) |

Exceptions:
- Grief interview overlay: `p-8` (32px) internal padding — slightly generous for the centered modal feel. Keep as-is.
- Archive sidebar width: fixed `w-64` (256px) — not a spacing token, but a layout constraint. Do not change.
- Container max-width: `max-w-5xl` (960px) — the demo viewport is 1280×800; this leaves 160px each side for the altar scroll visual. Keep as-is.
- SVG connector overlay height: 60px fixed — the incense smoke paths originate from a 60px tall SVG that sits between the header and the grid. This is not spacing in the standard sense; it is a structural layout slot.

Source: codebase audit of `app/layout.tsx`, `components/PathColumn.tsx`, `components/PathTreeView.tsx`

## Typography

| Role | Size | Weight | Line Height | Class Example |
|------|------|--------|-------------|---------------|
| Body (node content) | 14px (`text-sm`) | 400 (regular) | 1.625 (`leading-relaxed`) | `text-sm leading-relaxed` |
| Label (column headers, archive labels, button text) | 12px (`text-xs`) | 400 (regular) | 1.5 | `text-xs tracking-[0.4em] uppercase` |
| Heading (form section title, grief question) | 18px (`text-lg`) | 300 (light) | 1.625 (`leading-relaxed`) | `text-lg leading-relaxed` |
| Display (input form title "Begin", primary section headers) | 20px (`text-xl`) | 300 (light) | 1.2 | `text-xl font-light tracking-[0.2em] uppercase` |

Notes:
- Exactly 4 sizes: 12px, 14px, 18px, 20px.
- Exactly 2 weights: 300 (light) for headings and display text, 400 (regular) for body and labels.
- The "Cast Chart" submit button achieves visual hierarchy via `tracking-widest` + `uppercase` + `text-verse-gold` accent color — no third weight needed.
- `tabular-nums` Tailwind utility applies to all numeric display elements (depth indicator, question counter "Question 1 of 3"). Add via `font-variant-numeric: tabular-nums` or `font-feature-settings: "tnum"`. Apply class `tabular-nums` — already available in Tailwind v3. Source: RESEARCH.md UI-06.
- Vietnamese diacritics are supported by the Be Vietnam Pro `vietnamese` subset already loaded. No additional config needed.
- Letter-spacing: `tracking-[0.4em]` for column headers, `tracking-widest` for button labels and metadata text. These are pre-existing — maintain them.
- Do not introduce additional font sizes. The current 4 sizes cover all UI states in this phase.

Source: `components/InputForm.tsx`, `components/PathColumn.tsx`, `components/GriefInterviewOverlay.tsx`, `components/GriefArchiveSidebar.tsx` (codebase audit)

## Color

| Role | Hex | Tailwind Token | Usage |
|------|-----|----------------|-------|
| Dominant (60%) | `#1A0A00` | `verse-black` / `bg-verse-black` | Body background, input field backgrounds, node card backgrounds |
| Secondary (30%) | `#8B1A1A` | `verse-red` | Grief interview overlay panel (`bg-verse-red/90`), submit button background, section dividers |
| Accent (10%) | `#C9A84C` | `verse-gold` | Reserved-for list below |
| Ash state | `#6B6B6B` | `verse-ash` | Abandoned path column headers, depth dots on abandoned paths, abandoned path node borders |
| Destructive / error | `#F87171` | `red-400` (Tailwind default) | Form validation errors only — not a brand color |

Accent (`verse-gold`) reserved exclusively for:
1. Column header text on active (non-abandoned) paths
2. Depth indicator filled dots (`●`) on active paths
3. Active node left-border accent (`border-verse-gold/30`)
4. Live streaming node border (`border-verse-gold/40`)
5. Expand / "Go deeper" / "Reveal path" button border and text
6. SVG branch connector lines (`stroke="#C9A84C"`, opacity 0.4–0.6)
7. Input field borders on focus (`focus:border-verse-gold`)
8. Form label text (`text-verse-gold`)
9. Archive sidebar header "Paths Released"
10. Gold decorative rule below the page header (altar scroll top border)

Gold must NOT appear on: abandoned path elements (use `verse-ash`), body paragraph text (use `verse-paper/90`), error messages (use `red-400`), or secondary metadata (use `verse-paper/30–50`).

Paper text (`#F5ECD7`, `verse-paper`) is used for all body prose text at varying opacities:
- `verse-paper/90` — committed node content (high legibility)
- `verse-paper/70` — streaming/in-progress text (slightly dimmed)
- `verse-paper/50` — form helper text, subtitle
- `verse-paper/30–40` — metadata, secondary labels, disabled states

Ash state transition: When a path is abandoned, apply `transition-colors duration-700` to column header (`text-verse-gold/70` → `text-verse-ash/50`), depth dots (`text-verse-gold` → `text-verse-ash/60`), and node borders (`border-verse-gold/30` → `border-verse-ash/20`). Remove flat `opacity-40` on the wrapper div — replace with targeted per-element color transitions.

Source: `tailwind.config.ts`, `components/PathColumn.tsx`, REQUIREMENTS.md (UI-01, UI-05), RESEARCH.md (Pattern 5)

## Interaction States

### Button States

| State | Visual |
|-------|--------|
| Default | `border-verse-gold/30 text-verse-gold/70` |
| Hover | `border-verse-gold/60 text-verse-gold` |
| Disabled | `opacity-40 cursor-not-allowed` (no hover effect) |
| Focus | `ring-2 ring-verse-gold/50` (visible keyboard focus ring) |

### Input States

| State | Visual |
|-------|--------|
| Default | `border border-verse-gold/40` |
| Focus | `border-verse-gold ring-1 ring-verse-gold/50` |
| Error | `border-red-400/60` (implied — error message appears below) |

### Streaming Cursor

A blinking `|` character (`animate-pulse`) appears inline at the end of streaming node text. Remove it when `isStreaming` transitions to `false`. The cursor uses `ml-0.5` spacing.

### Abandoned Path

Full column fades via per-element color transitions (duration 700ms). No dimming overlay. The column remains in its grid position — it does not collapse or hide.

### Grief Interview Overlay

- Backdrop: `bg-black/70` fixed overlay
- Panel: `bg-verse-red/90 border border-verse-gold/30` — sharp-cornered box, no border-radius
- Entrance: fade in `opacity: 0 → 1` over 300ms
- Not dismissible — user must complete or the session state has no close mechanism

### SVG Branch Connectors (new in Phase 4)

- Initial state: `pathLength: 0, opacity: 0`
- Animate to: `pathLength: 1, opacity: 0.6`
- Duration: 1.4s, easing `[0.25, 0.46, 0.45, 0.94]` (easeOut cubic)
- Stroke: `#C9A84C` (verse-gold), width 1px, `strokeLinecap="round"`, `strokeOpacity` via animate not fill
- SVG dimensions: `width="100%" height="60" viewBox="0 0 960 60" preserveAspectRatio="none"`
- Three bezier curves: center-top `(480, 0)` → each column center: `(160, 60)`, `(480, 60)`, `(800, 60)` — using cubic bezier control points for smoke-curve shape
- Trigger: on mount when PathTreeView first renders with a valid chart

Source: RESEARCH.md (Pattern 1), REQUIREMENTS.md (UI-04)

### Paper Texture (new in Phase 4)

- Implementation: CSS SVG data URI in `app/globals.css` on `body`
- Pattern: `feTurbulence` fractalNoise, `baseFrequency="0.65"`, `numOctaves="3"`, `stitchTiles="stitch"`, opacity `0.05`
- Tile size: `background-size: 200px 200px`
- Does not affect text legibility at opacity 0.05

Source: RESEARCH.md (Pattern 2)

### Altar Scroll Layout (refined in Phase 4)

- Body: `bg-verse-black` + paper texture overlay
- Container: `max-w-5xl mx-auto px-4 py-8` (existing — keep)
- Top decorative rule: 1px horizontal line below the page header, `border-verse-gold/20`
- Container sides: box-shadow `inset 0 0 60px rgba(139,26,26,0.08)` — subtle red-lacquer ambient glow on inner edges, applied to the `<main>` element
- No border-radius anywhere — lacquerware aesthetic is rectilinear

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (form submit) | "Cast Chart" — already implemented, keep as-is |
| Expand button (depth 0, first reveal) | "Reveal path" — keep as-is; tender, not instructional |
| Expand button (depth 1–4) | "Go deeper" — keep as-is |
| Path complete state | "Path complete" — keep as-is |
| Abandon button | "Abandon" — keep as-is; plain, honest, not dramatic |
| Grief interview final button | "Release this path" — keep as-is; grounded, not ceremonial |
| Grief interview continue button | "Continue" — keep as-is |
| Archive sidebar header | "Paths Released" — keep as-is |
| Archive empty state | "No paths have been released yet." — keep as-is |
| Abandoned path note | "Path abandoned" — keep as-is |
| Form section title | "Begin" — keep as-is |
| Form subtitle | "Your chart will be cast from these details." — keep as-is |
| Form field: life decision placeholder | "Describe the crossroads. What are you trying to decide?" — keep as-is |
| Form field: birthplace placeholder | "e.g. Hà Nội, Ho Chi Minh City" — keep as-is |
| Depth counter (grief overlay) | "Question {N} of 3" — keep as-is |
| Error state (form validation) | Field-level inline errors from Zod — keep existing pattern; no global error banner needed |
| API failure (fallback trigger) | Silent — no user-visible message. Fallback JSON loads transparently. The user sees node content appear as if it streamed normally. |
| Path at depth 5 (max depth reached) | "Path complete" — already covers this state |

Tone guidelines (UI-07):
- All copy must be declarative and grounded. No mystical metaphors ("the stars reveal..."), no gamification ("Level up!", "Achievement unlocked").
- Questions in the grief interview are the only copy that should carry emotional weight — and they do this through plainness, not decoration.
- All button labels are lowercase except when ALL CAPS tracking-widest style is applied (column headers, labels). Buttons use uppercase + widest tracking as a design pattern, not shouting.
- Do not add exclamation marks anywhere.

Destructive actions in this phase:
- "Abandon" path: The abandon button itself is the confirmation. No secondary confirm dialog. The grief interview is the ritual gate — answering all 3 questions IS the confirmation. If user abandons mid-interview and closes the tab, the path is not abandoned (no partial state written). This is correct behavior.
- `?reset` URL parameter: Clears ALL session state including grief archive. No UI confirmation shown. This is a demo tool, not a user-facing feature — it is triggered deliberately by the presenter typing the URL. Acceptable for V1.

Source: codebase audit of all components, REQUIREMENTS.md (UI-07), RESEARCH.md (Pattern 5), ROADMAP.md Phase 4

## Animation Contract

All animations use the `motion` package (`import { motion, AnimatePresence } from 'motion/react'`). Never import from `framer-motion`.

| Animation | Trigger | Duration | Easing | Notes |
|-----------|---------|----------|--------|-------|
| Node entrance | New node committed to store | 400ms | default | `initial: { opacity: 0, y: 12 }` → `animate: { opacity: 1, y: 0 }` |
| Node exit (AnimatePresence) | Node removed (not expected in V1) | 200ms | default | `exit: { opacity: 0 }` |
| Branch connector draw | PathTreeView mount with valid chart | 1400ms | `[0.25, 0.46, 0.45, 0.94]` | `pathLength: 0 → 1` on `motion.path` |
| Grief overlay entrance | `setActiveInterview(pathId)` | 300ms | default | `opacity: 0 → 1` |
| Archive panel entrance | Toggle open | 300ms | default | `opacity: 0, x: 20 → opacity: 1, x: 0` |
| Ash color transition | `isAbandoned` flips to true | 700ms | CSS `transition-colors` | Applied via Tailwind on individual elements |

## Resolution Contract

Target: 1280×800 (projector standard, DEMO-03).

- `max-w-5xl` (960px) container centered in 1280px viewport — 160px padding each side. No overflow expected.
- Archive sidebar `w-64` (256px) — total layout: 960px grid + 256px sidebar + 24px gap = 1240px at full width. Fits in 1280px.
- Grief overlay: `max-w-md w-full mx-4` — 448px wide, centered. Fits at 1280×800 with room.
- Node text `text-sm leading-relaxed` — readable at 1280px without overflow.
- No horizontal scroll must occur at 1280×800. The `min-w-0` on the flex-1 column grid wrapper is critical to prevent overflow — verify it is present in PathTreeView.
